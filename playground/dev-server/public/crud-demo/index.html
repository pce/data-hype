Hype — CRUD demos Markup-first examples (table, list, swipe)

A small collection of markup-first CRUD demos. Each demo intentionally contains minimal JavaScript: pages declare intent with data attributes and templates while the Hype runtime and optional plugins wire behavior. These pages are designed to be CSP-friendly — styles are external and demo behavior is provided by the runtime.

Available CRUD demos

Demos

  • Table — classic rows & actions
    A table-driven CRUD example using a declarative create form and a template for rows.
  • List — stacked list rendering
    A stacked list demo (card-style) rendered from a template; markup-first and accessible.
  • Swipe — horizontal card carousel
    Scroll-snap cards intended for touch interactions; Hype/datatable renders the cards from a template.

Developer notes

  1. Ensure an ESM Hype runtime is available at /static/js/hype.js (your build step should produce this).
  2. The demos rely on a small, CSP-friendly bootstrap module at /static/js/crud-demo-bootstrap.js which imports the runtime and calls hype.init() and plugin autowiring when available.
  3. Tailwind and the shared demo stylesheet
    4. live at /tailwind.css and /static/css/crud-demo.css. Keep styles external for strict CSP setups.
  4. Run the dev environment locally (example): ./dev.sh start. If you see 404s for the runtime or bootstrap, build the frontend artifacts and ensure they are copied into the backend public directory.

Markup-first guidance: prefer declarative attributes such as

data-hype-crud="items", templates for rendering, and keep demo JS minimal so the runtime can autowire behavior. This keeps examples DRY and easy to maintain.
