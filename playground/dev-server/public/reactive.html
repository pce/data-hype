Available Directives

data-hype-state

Declares reactive state for the component. Must be valid JSON or JavaScript object notation.

data-hype-state='{ "open": false, "count": 0 }'

data-hype-on-{event}

Handles events with access to state and helpers (toggle, set, $fetch, $state, $el). Prefer DSL JSON for CSP-safe evaluation: ['toggle','open']. The runtime may also accept the call-style string in dev, but the DSL JSON form is recommended.

data-hype-on-click='["toggle","open"]' (preferred: DSL JSON)

data-hype-show

Shows/hides elements based on expression evaluation. Use DSL JSON for complex logic: e.g. [\">\", [\"get\", \"count\"], 0] or simple identifiers like open.

data-hype-show='[">", ["get", "count"], 0]'

data-hype-class-{className}

Conditionally applies CSS classes based on expression.

data-hype-class-text-red-500="isActive"

data-hype-bind-{attribute}

Binds attribute values to state expressions.

data-hype-bind-disabled="loading"
